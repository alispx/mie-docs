Getting Started

Welcome

Mie Theme Customizer Customizer is a simple framework that consists of a series of classes and functions to produce an incredibly simple to use customizer framework. This is a library to in include directly into your next WordPress theme.

With all the changes in WordPress lately of the preferred method of customizing a theme now shifting to being through the Customizer, we thought it best to create an eay to impliment framework calledd Mie (pronounced "My"). Our main goal is to produce a customizer that any developer can get started right away and end up with beautiful results.

Mie is created by a high level developer with one thing in mind - to help the new developer build with ease

How to use Mie

To get Mie working on your theme, simply embed the code below into your theme functions.php file.

define( 'MIE_DIR', trailingslashit( get_template_directory() . '/customizer' ) );
define( 'MIE_URI', trailingslashit( get_template_directory_uri() . '/customizer' ) );
require_once( get_template_directory() . '/customizer/mie-customizer.php' );
require_once( get_template_directory() . '/customizer/customizer-data.php' );

Kemudian hanya tinggal merubah isi dari file customizer-data.php sesuai dengan kebutuhan. Mie customizer options are all stored in /customizer/customizer-data.php

Make Mie life so easy

The true benefits of using Mie are quite obviouis.

  • Easy Configuration
  • Live Preview
  • Auto Generate CSS

Getting Started with Mie Customizer

Create Panel

Penel ini merupakan sebuah element paling atas yang dapat menampung section-section untuk mengorganisir setting agar tampak teratur. Panel ini baru ada pada WordPress 4.0 bersamaan dengan di rilisnya Widget Customizer. Untuk membuat panel, silahkan isi type dengan 'panel' seperti contoh berikut

$mie_options[] = array(
	'slug'		=> 'mie_standard_Option', // panel id
	'label'		=> __( 'Standard Settings', 'mie' ), // panel name
	'priority'	=> 10,
	'type' 		=> 'panel'
);

Create Section

Section adalah wadah untuk menampung kontrol-kontrol per masing2 element. Section dapat berdiri sendiri tanpa memerlukan panel. Untuk membuat sebuah section maka type harus di isi dengan 'section'. apabila anda telah membuat sebuah section dan belum tampil maka kemungkinan section tersebut masih kosong, belum ada control-control yang di assign ke section tersebut

$mie_options[] = array(
	'slug'		=> 'mie_header_settings', //Section id
	'label'		=> __( 'Header Settings', 'mie' ), //section name
	'panel' 	=> 'mie_standard_Option', //Panel id where this section will be hooked
	'priority'	=> 10,
	'type' 		=> 'section'
);

Color

$mie_options[] = array( 
	'slug'		=> 'mie_header_background', //Control id
	'default'	=> '#fff', //default value
	'priority'	=> 1, //priority
	'label'		=> __( 'Header Background', 'mie' ), //Control name
	'section'	=> 'mie_header_settings', //section name
	'selector'	=> '.site-header', //CSS selector
	'property'	=> 'background', //CSS property
	'type' 		=> 'color' //Control type
);

Bentuk output dari type 'color' ini adalah sebagai berikut : 

selector { property : newvalue property2 }
untuk property2 sifatnya optional.

Color RGB

$mie_options[] = array( 
	'slug'		=> 'mie_example_color_rgb', 
	'default'	=> '#250D07', 
	'priority'	=> 1, 
	'label'		=> __( 'RGB Color', 'mie' ),
	'section'	=> 'mie_color_settings',
	'selector'	=> '.site-header',
	'property'	=> 'background',
	'property2'	=> '0.5', //Color Opacity
	'type' 		=> 'color_rgb'
);

Bentuk output dari type 'color_rgb' sebagai berikut : 

selector { property : rgb(newvalue, property2) }

dalam hal ini property2 diisi dengan nilai opacity

Dependency Field

Fitur ini masalah dalam pengembangan, saat ini yang sudah dapat digunakan yaitu field checkbox yang digunakan sebagai dependency. yaitu dengan membaca nilai dari field checkbox true atau false. formatnya sebagai berikut : 

$mie_options[] = array( 
	'slug'		=> 'mie_example_dependency', 
	'default'	=> '', 
	'priority'	=> 2, 
	'label'		=> __( 'Dependency Field', 'mie' ),
	'section'	=> 'mie_checkbox_settings',
	'selector'	=> '.site-header',
	'type' 		=> 'text',
	'dependson' => 'mie_example_checkbox_field',
);
			
'dependson' => 'mie_example_checkbox_field',
array 'dependson' diisi dengan slug dari field yang di rujuk.

Display Customizer Data

Untuk menampilkan data dari customizer yang telah kita buat maka kita dapat menggunakan fungsi bawaan dari wordpress get_theme_mod(). formatnya seperti berikut ini :

echo get_theme_mod( $slug );
//contoh :
echo get_theme_mod( 'mie_header_background' );

ada beberapa field yang outputnya akan di generate secara otomatis seperti field : color, color_rgb. untuk type field lain silahkan anda gunakan seperlunya
